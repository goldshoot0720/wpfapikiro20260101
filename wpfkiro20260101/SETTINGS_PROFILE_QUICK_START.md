# 設定檔管理 - 快速開始指南

## 🚀 功能簡介

設定檔管理功能讓您可以儲存、載入和管理多達 100 筆應用程式設定檔。每個設定檔包含完整的後端服務配置，讓您輕鬆在不同環境間切換。

## 📋 主要特色

- ✅ 支援最多 100 筆設定檔
- ✅ 支援所有後端服務 (Appwrite, Supabase, NHost 等)
- ✅ 一鍵儲存當前設定
- ✅ 一鍵載入已儲存設定
- ✅ 匯入/匯出 JSON 格式
- ✅ 搜尋和篩選功能
- ✅ 本地和雲端同步儲存

## 🎯 快速使用步驟

### 1. 開啟設定檔管理
1. 進入「系統設定」頁面
2. 找到「設定檔管理」區域
3. 點擊「📁 管理設定檔」按鈕

### 2. 儲存當前設定
1. 在設定檔管理視窗中點擊「💾 儲存當前設定」
2. 輸入設定檔名稱（例如：「生產環境」、「測試環境」）
3. 輸入描述（選填）
4. 點擊「儲存」

### 3. 載入設定檔
1. 在設定檔列表中選擇要載入的設定檔
2. 點擊「📂 載入設定檔」
3. 確認載入操作
4. 重新啟動應用程式（建議）

### 4. 管理設定檔
- **編輯**: 選擇設定檔 → 點擊「✏️ 編輯設定檔」
- **刪除**: 選擇設定檔 → 點擊「🗑️ 刪除設定檔」
- **搜尋**: 在搜尋框中輸入關鍵字

### 5. 匯入/匯出
- **匯出全部**: 點擊「📤 匯出設定檔」
- **匯出選中**: 選擇設定檔 → 點擊「📤 匯出選中」
- **匯入**: 點擊「📥 匯入設定檔」→ 選擇 JSON 檔案

## 💡 使用建議

### 設定檔命名
- 使用清楚的名稱，例如：
  - `生產環境 - Appwrite`
  - `測試環境 - Supabase`
  - `開發環境 - 本地MySQL`

### 描述內容
- 記錄設定檔的用途和特殊說明
- 例如：「正式上線環境，包含完整 API 權限」

### 備份策略
- 定期匯出重要設定檔
- 將匯出的 JSON 檔案儲存在安全位置
- 建議使用版本控制管理設定檔

## ⚠️ 注意事項

### 安全性
- 設定檔包含 API Key 等敏感資訊
- 匯出的 JSON 檔案請妥善保管
- 不要將包含敏感資訊的檔案上傳到公開位置

### 限制
- 最多支援 100 筆設定檔
- 設定檔名稱最大 100 字元
- 描述最大 500 字元

### 相容性
- 支援不同版本間的設定檔匯入
- 自動處理缺失欄位的預設值
- 匯入時自動處理名稱衝突

## 🔧 故障排除

### 常見問題

**Q: 設定檔載入後沒有生效？**
A: 請重新啟動應用程式以確保所有設定生效。

**Q: 無法儲存設定檔？**
A: 檢查是否已達到 100 筆限制，或設定檔名稱是否重複。

**Q: 匯入設定檔失敗？**
A: 確認 JSON 檔案格式正確，且不超過數量限制。

**Q: 找不到之前儲存的設定檔？**
A: 檢查搜尋框是否有輸入篩選條件。

### 錯誤恢復
- 如果本地檔案損壞，可從 Appwrite 雲端恢復
- 支援手動編輯 JSON 檔案進行修復
- 可以重置所有設定檔重新開始
